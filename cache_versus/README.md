# Сравнение стратегий кеширования

Бенчмарк трёх стратегий кеширования — **Lazy Loading**, **Write-Through** и **Write-Back** — в трёх сценариях нагрузки с автоматической генерацией Markdown-отчёта.

---

## Стратегии

| Стратегия | Чтение | Запись |
|-----------|--------|--------|
| **Lazy Loading / Cache-Aside** | Кеш → при промахе БД → заполнить кеш | Только в БД, кеш инвалидируется |
| **Write-Through** | Кеш → при промахе БД → заполнить кеш | Синхронно в БД и кеш |
| **Write-Back** | Кеш → при промахе БД → заполнить кеш | Только в кеш, в БД — асинхронно |

## Быстрый старт

```bash
make run
```

Команда собирает Docker-образ приложения, поднимает Redis и PostgreSQL в той же сети,
запускает бенчмарк и останавливает контейнеры по завершении.
Отчёт сохраняется в `./reports/` на хосте через volume-mount.

Для очистки томов:

```bash
make down
```

## Переменные окружения

| Переменная | Значение по умолчанию | Описание |
|------------|-----------------------|----------|
| `REDIS_ADDR` | `localhost:6379` | Адрес Redis |
| `REDIS_PASSWORD` | _(пусто)_ | Пароль Redis |
| `POSTGRES_DSN` | `postgres://cache_user:cache_pass@localhost:5432/cache_db?sslmode=disable` | DSN PostgreSQL |

Параметры бенчмарка задаются в [config/config.go](config/config.go):

| Параметр | По умолчанию | Описание |
|----------|-------------|----------|
| `Duration` | `30s` | Длительность одного сценария |
| `Concurrency` | `50` | Параллельных воркеров |
| `TotalKeys` | `1000` | Размер ключевого пространства |
| `CacheTTL` | `5m` | TTL записи в кеше |
| `WriteBackFlushInterval` | `2s` | Интервал сброса Write-Back в БД |

## Сценарии нагрузки

| Сценарий | Чтений | Записей |
|----------|--------|---------|
| read-heavy | 80% | 20% |
| balanced | 50% | 50% |
| write-heavy | 20% | 80% |

Каждая из трёх стратегий прогоняется по всем трём сценариям → итого **9 тестов**.

---

## Собираемые метрики

| Метрика | Описание |
|---------|----------|
| **Throughput** | Операций в секунду (req/s) |
| **Средняя задержка** | Среднее время одной операции (мс) |
| **P99 задержка** | 99-й перцентиль задержки (мс) |
| **Обращений в БД** | Количество реальных запросов к PostgreSQL |
| **Hit Rate кеша** | Доля чтений, обслуженных из кеша (%) |

## Отчёт

После каждого запуска в папке `reports/` автоматически создаётся файл вида:

```
reports/report_2025-04-29_14-30-00.md
```

Отчёт содержит:
- конфигурацию теста
- сводную таблицу результатов по всем 9 прогонам
- анализ победителей по каждому сценарию
- выводы о применимости каждой стратегии

## Тесты

Запустить юнит-тесты (без инфраструктуры):

```bash
go test ./...
```

Покрытые пакеты:

| Пакет | Что тестируется |
|-------|----------------|
| `internal/metrics` | throughput, hit rate, средняя и P99 задержка |
| `internal/strategy` | чтение с хитом/промахом, запись, инвалидация, сброс Write-Back |

## Выводы

| Стратегия | Лучше всего подходит |
|-----------|---------------------|
| **Lazy Loading** | Нагрузка с преобладанием чтений; допустима небольшая устаревшесть |
| **Write-Through** | Смешанная нагрузка; важна строгая согласованность |
| **Write-Back** | Нагрузка с преобладанием записей; допустим риск потери при аварии |
